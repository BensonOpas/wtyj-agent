## Unexpected findings

**Conftest fixture required, not in the brief.** Adding the strict-mode allowlist block to `clients/bluemarlin/config/client.json` immediately broke 13 existing tests in `test_138_dm_booking.py`, `test_143_zernio_whatsapp.py`, `test_186_channel_adapters.py`, `test_187_sender_registry.py` because `wtyj/tests/conftest.py` points `CLIENT_CONFIG_PATH` at BlueMarlin's config and those tests call production code paths (`ZernioSender.send`, `_process_zernio_event`) without mocking the new guard. Fix: added an autouse fixture in `wtyj/tests/conftest.py` that pops `channel_account_allowlist` from `config_loader._cache` for the duration of each test (and restores after). Required because `config_loader.get_raw()` returns `dict(_load())` — a shallow copy — so popping from the copy doesn't affect what subsequent calls see; you have to mutate `_cache` directly. The Brief 238 tests aren't affected by the strip because they patch `shared.config_loader.get_raw` to return their own fake_cfg, which overrides the strip at function-call level. The brief-reviewer didn't flag this regression because reviewer doesn't run tests; future briefs that add a config field read by production code should grep for `CLIENT_CONFIG_PATH` users and assess test impact up front.
