## Context

`hello@unboks.org` is the new public contact email for the unboks tenant — calvin-csa needs to poll it (read incoming prospect emails) and reply via SMTP. The mailbox is on **Google Workspace** (SR pays for the seats). Our existing `email_adapter.py` only knows Microsoft Azure OAuth → `outlook.office365.com` IMAP + `smtp.office365.com` SMTP. Calvin-csa can't reach a Gmail mailbox today.

After a /scope check we cut the heavy plan (provider abstraction with `EMAIL_PROVIDER` switch, OAuth client setup, JWT-signed service account, domain-wide delegation, interactive bootstrap script) and chose the minimum-viable path: **Gmail app password** (16-character secret generated by 2-Step-Verification-enabled accounts at `myaccount.google.com/apppasswords`). The 16-char password was generated by Benson 2026-05-05 for `hello@unboks.org`.

App passwords for Gmail IMAP + SMTP are officially supported (with 2FA enabled) and documented as the intended path for IMAP clients that can't do OAuth. Workspace admins CAN disable them centrally — if that happens later we pivot to OAuth/service-account, but we don't preemptively build for a problem we don't have.

### Current code (`wtyj/agents/marina/email_adapter.py`)

- Line 32: `IMAP_HOST = "outlook.office365.com"`
- Line 34: `SMTP_HOST = "smtp.office365.com"`
- Line 99: `imap_connect()` calls `oauth_token(...)` → `IMAP4_SSL(IMAP_HOST, IMAP_PORT)` → `im.authenticate("XOAUTH2", ...)`
- Line 108: `smtp_send(...)` calls `oauth_token(...)` → `SMTP(SMTP_HOST, SMTP_PORT)` → `s.docmd("AUTH", "XOAUTH2 " + auth_b64)`

Both functions are hardcoded to Microsoft. Both need a Gmail branch.

### Why one env-var as the switch

`EMAIL_PASSWORD` env var presence IS the switch. No `EMAIL_PROVIDER=google` needed — if a password is set, it can only mean Gmail (Microsoft's tenancy doesn't use static passwords for IMAP/SMTP, all enterprise OAuth). One signal, one branch.

This means:
- BlueMarlin's `platform.env`: NO `EMAIL_PASSWORD` → existing OAuth path runs unchanged.
- Adamus's `platform.env`: NO `EMAIL_PASSWORD` → existing OAuth path runs unchanged (still pending bootstrap separately, not this brief's concern).
- unboks's `platform.env`: `EMAIL_PASSWORD=<16chars>` set → Gmail LOGIN auth runs.

Single env var, zero risk to BlueMarlin/Adamus.

### Out of scope

- **OAuth + service account + domain-wide delegation paths.** Not needed today; can be added later if Google revokes app passwords or workspace policy changes.
- **Interactive bootstrap script** (`bootstrap_google_token.py`). Skipped — app passwords don't need browser-based bootstrap.
- **Adamus's still-pending Microsoft OAuth bootstrap** for `sophia@wetakeyourjob.com`. Different tenant, different provider, different ticket.
- **SMTP-host override via env var.** Hardcoding `smtp.gmail.com:587` and `imap.gmail.com:993` for the Gmail branch is fine. If we ever need a non-Gmail-but-also-non-Microsoft provider, that's a future expansion.

## Why This Approach

**Considered alternatives:**

1. **Full provider abstraction** with `EMAIL_PROVIDER=microsoft|google` switch + OAuth client + token refresh + bootstrap script. Rejected per /scope — enterprise-grade for ~10 leads/month early on. Wrong scope for this stage.

2. **Service account with domain-wide delegation.** Cleanest enterprise pattern, but requires SR's intervention in `admin.google.com` to grant delegation. Also requires JWT signing at runtime. ~2x the code of the app password path. Rejected for now; revisit if app passwords get blocked.

3. **OAuth client + admin pre-approval.** Requires running a one-time interactive flow per mailbox. App password is simpler. Rejected.

4. **Chosen: App password with single env-var switch.** ~15 lines of code change. No new files. No bootstrap. No OAuth client. No service account. If Google revokes or admin disables, we know in 30 seconds and pivot. Until proven blocked, this is right.

**Tradeoff:** App passwords are sometimes considered "less secure" (Google's framing for the deprecated less-secure-apps feature was different — that was IMAP-with-account-password; app-passwords-with-2FA is a separately maintained mechanism). Risk profile:
- Today: works
- 6 months: probably still works
- 2 years: maybe Google forces OAuth-only, we'd need to add OAuth path then

For the early sales-push stage, today-velocity > future-migration-risk. Acceptable.

**Tradeoff:** No SMTP-host configurability. If a future tenant uses a non-Gmail-non-Microsoft provider (Mailgun, Fastmail, custom IMAP), we'd need to genuinely abstract. Today: not happening. YAGNI.

```python
def imap_connect():
    """Brief 204: Gmail app password mode when EMAIL_PASSWORD is set; else
    Microsoft OAuth XOAUTH2 (existing path)."""
    if EMAIL_PASSWORD:
        # Gmail app password — basic LOGIN auth. Strip whitespace because
        # Google formats app passwords as 4 groups of 4 chars separated by
        # spaces; both formats work for IMAP but we strip for consistency.
        password = EMAIL_PASSWORD.replace(" ", "")
        im = imaplib.IMAP4_SSL(GMAIL_IMAP_HOST, IMAP_PORT)
        im.login(EMAIL_ADDR, password)
        return im
    # Existing Microsoft OAuth path
    token = oauth_token("offline_access https://outlook.office.com/IMAP.AccessAsUser.All")
    auth_string = f"user={EMAIL_ADDR}\x01auth=Bearer {token}\x01\x01".encode("utf-8")
    im = imaplib.IMAP4_SSL(IMAP_HOST, IMAP_PORT)
    im.authenticate("XOAUTH2", lambda _: auth_string)
    return im
```

**Step 4.** Modify `smtp_send()` (line 108) similarly. The function's body builds the MIMEMultipart message — that part stays. The auth + connect part forks:

```python
def smtp_send(to_addr: str, subject: str, body: str, in_reply_to=None, references=None, reply_to=None):
    # Build the message (unchanged)
    msg = MIMEMultipart()
    msg["From"] = "Marina <{}>".format(EMAIL_ADDR)
    msg["To"] = to_addr
    msg["Subject"] = subject
    msg["Message-ID"] = make_msgid(domain=EMAIL_ADDR.split("@")[1])
    if in_reply_to:
        msg["In-Reply-To"] = in_reply_to
    if references:
        msg["References"] = references
    if reply_to:
        msg["Reply-To"] = reply_to
    msg.attach(MIMEText(body, "plain", "utf-8"))

    if EMAIL_PASSWORD:
        # Brief 204: Gmail app password — basic LOGIN auth via STARTTLS.
        password = EMAIL_PASSWORD.replace(" ", "")
        s = smtplib.SMTP(GMAIL_SMTP_HOST, SMTP_PORT, timeout=30)
        s.ehlo(); s.starttls(); s.ehlo()
        s.login(EMAIL_ADDR, password)
        s.sendmail(EMAIL_ADDR, [to_addr], msg.as_string())
        s.quit()
        return

    # Existing Microsoft OAuth XOAUTH2 path
    token = oauth_token("offline_access https://outlook.office.com/SMTP.Send")
    auth_string = f"user={EMAIL_ADDR}\x01auth=Bearer {token}\x01\x01"
    auth_b64 = base64.b64encode(auth_string.encode("ascii")).decode("ascii")
    s = smtplib.SMTP(SMTP_HOST, SMTP_PORT, timeout=30)
    s.ehlo(); s.starttls(); s.ehlo()
    code, resp = s.docmd("AUTH", "XOAUTH2 " + auth_b64)
    if code != 235:
        s.quit()
        raise RuntimeError(f"SMTP AUTH failed: {code} {resp!r}")
    s.sendmail(EMAIL_ADDR, [to_addr], msg.as_string())
    s.quit()
```

"""Brief 204: Gmail app-password auth path in email_adapter.py."""

import os

# Match established test pattern
os.environ.setdefault("DASHBOARD_PASSWORD", "testpass")

from unittest.mock import MagicMock, patch


@patch("agents.marina.email_adapter.imaplib.IMAP4_SSL")
def test_imap_connect_uses_gmail_login_when_password_set(mock_imap_class):
    """When EMAIL_PASSWORD is set, imap_connect connects to imap.gmail.com
    and uses LOGIN auth (no OAuth call)."""
    # Patch the module-level constants AFTER import — module-level os.environ.get
    # at import time captures empty defaults; we rebind the EMAIL_PASSWORD module
    # attribute directly to simulate "env var was set."
    import agents.marina.email_adapter as adapter

    mock_im = MagicMock()
    mock_imap_class.return_value = mock_im

    with patch.object(adapter, "EMAIL_PASSWORD", "abcdwxyz12345678"), \
         patch.object(adapter, "EMAIL_ADDR", "hello@unboks.org"):
        result = adapter.imap_connect()

    # Connected to Gmail
    mock_imap_class.assert_called_once_with("imap.gmail.com", 993)
    # LOGIN auth (not XOAUTH2)
    mock_im.login.assert_called_once_with("hello@unboks.org", "abcdwxyz12345678")
    # XOAUTH2 path NOT taken
    mock_im.authenticate.assert_not_called()
    assert result is mock_im


@patch("agents.marina.email_adapter.imaplib.IMAP4_SSL")
@patch("agents.marina.email_adapter.oauth_token")
def test_imap_connect_uses_microsoft_oauth_when_no_password(mock_oauth, mock_imap_class):
    """When EMAIL_PASSWORD is empty (BlueMarlin's case), imap_connect uses the
    existing Microsoft OAuth XOAUTH2 path."""
    import agents.marina.email_adapter as adapter

    mock_oauth.return_value = "fake-access-token"
    mock_im = MagicMock()
    mock_imap_class.return_value = mock_im

    with patch.object(adapter, "EMAIL_PASSWORD", ""), \
         patch.object(adapter, "EMAIL_ADDR", "hello@wetakeyourjob.com"):
        result = adapter.imap_connect()

    # Connected to Outlook (NOT Gmail)
    mock_imap_class.assert_called_once_with("outlook.office365.com", 993)
    # OAuth was called
    mock_oauth.assert_called_once()
    # XOAUTH2 auth (not LOGIN)
    mock_im.authenticate.assert_called_once()
    args, _ = mock_im.authenticate.call_args
    assert args[0] == "XOAUTH2"
    # LOGIN path NOT taken
    mock_im.login.assert_not_called()
    assert result is mock_im

3. **SMTP Gmail mode** — verifies the new SMTP branch: connects to `smtp.gmail.com:587`, runs the STARTTLS dance, calls `LOGIN`, does NOT use the XOAUTH2 docmd path. Catches host-constant copy-paste errors.
